---
title: "Embedded Signing Quickstart"
description: "Create your first signature request and get an embeddable signing URL."
---

This quickstart takes about **10 minutes**. In just **4 API calls**, you go from a plain PDF or DOCX to a signed document with an audit trail.

## Prerequisites

Before you begin, you need:

- An **API key** from [Settings](https://app.formabledocs.com/settings) — see [Set up a developer account](/developer-account) and [Authentication](/authentication)
- A **PDF or DOCX file** under 40MB to send for signature

<Info>
  The examples below use the production base URL `https://api.formabledocs.com/api`. Replace `$TOKEN` with your API key (it starts with `fmbl_`).
</Info>

## Steps

<Steps>
  <Step title="Upload a template">
    Upload your document as `multipart/form-data`. Formable returns a `templateId` you'll use to send the document.

    ```bash
    curl --request POST \
      --url https://api.formabledocs.com/api/v0/templates \
      --header "Authorization: Bearer $TOKEN" \
      --form 'file=@./agreement.pdf' \
      --form 'filename=agreement.pdf'
    ```

    ```json Response
    {
      "templateId": "abc123xyz",
      "editTemplateAccess": {
        "editUrl": "https://app.formabledocs.com/template-setup/abc123xyz",
        "expiresAt": "2024-01-16T10:30:00.000Z"
      }
    }
    ```

    <Tip>
      Open the `editUrl` in a browser to place signature, text, and date fields on the document. See [Embedded templates](/walkthroughs/embedded-templates).
    </Tip>

    <Frame caption="Place a Signature field on the document, then click Save template.">
      <img
        src="/static/images/template/template-with-signature-field.png"
        alt="Template editor with a Signature field selected over the Customer line, showing Required toggle and Reference ID in the sidebar"
      />
    </Frame>
  </Step>

  <Step title="Create a signature request">
    Create a signature request from the template. Provide the `signer` (who signs) and the `sender` (who's sending it).

    ```bash
    curl --request POST \
      --url https://api.formabledocs.com/api/v0/signature-requests \
      --header "Authorization: Bearer $TOKEN" \
      --header 'Content-Type: application/json' \
      --data '{
        "templateId": "abc123xyz",
        "signer": { "email": "signer@example.com", "name": "Jane Doe" },
        "sender": { "email": "you@yourcompany.com", "name": "Your Company" }
      }'
    ```

    ```json Response
    {
      "signatureRequestId": "sr_456def"
    }
    ```
  </Step>

  <Step title="Generate a signing URL">
    Turn the signature request into an embeddable signing URL. The URL expires one hour after creation.

    ```bash
    curl --request POST \
      --url https://api.formabledocs.com/api/v0/signature-requests/sr_456def/url \
      --header "Authorization: Bearer $TOKEN"
    ```

    ```json Response
    {
      "signingUrl": "https://app.formabledocs.com/sign/embedded/xyz789abc",
      "expiresAt": "2024-01-16T10:30:00.000Z"
    }
    ```

    Load the `signingUrl` in an iframe or redirect the signer to it (link does not need to be embedded in iframe to be opened):

    ```html
    <iframe src="https://app.formabledocs.com/sign/embedded/xyz789abc" width="100%" height="800"></iframe>
    ```

    <Frame caption="The signer clicks Start and completes each required field on the document.">
      <img
        src="/static/images/embedded-signing/signing-start.png"
        alt="Embedded signing view showing document.docx with a Sign here field on the Customer line, a Required callout, and a Start button"
      />
    </Frame>
  </Step>

  <Step title="Retrieve the signed document">
    Once the signer completes the document, download the signed PDF from a presigned URL.

    ```bash
    curl --request GET \
      --url https://api.formabledocs.com/api/v0/signature-requests/sr_456def/signed-envelope \
      --header "Authorization: Bearer $TOKEN"
    ```

    ```json Response
    {
      "signedEnvelopePresignedUrl": "https://s3.amazonaws.com/bucket/signed-envelope.pdf?..."
    }
    ```

    <Note>
      This endpoint returns `409` until the document is signed. Poll [signature request events](/api-reference/endpoint/get-signature-request-events) or check the [signature request](/api-reference/endpoint/get-signature-request) `status` to know when it's `Completed`.
    </Note>

    The signed PDF includes an audit trail with created, sent, signed, and completed events.

    <Frame caption="Every signed document includes an audit trail with timestamps and actors.">
      <img
        src="/static/images/embedded-signing/signed-doc-with-audit-trail.png"
        alt="Formable audit trail for document.docx showing Created, Sent, Signed, and Completed events with status Completed"
      />
    </Frame>
  </Step>
</Steps>
