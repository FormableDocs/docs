---
title: "Authentication"
description: "Authenticate API requests with an API key from Settings."
---

The Formable API authenticates requests with a Bearer token in the `Authorization` header. Create an API key in [Settings](https://app.formabledocs.com/settings), then send it on every request.

```bash
Authorization: Bearer fmbl_YOUR_API_KEY
```

## Get an API key

API keys belong to an organization. Create one in the Formable app, then copy the key when it's shown. For screenshots of each step, see [Set up a developer account](/developer-account).

<Steps>
  <Step title="Create an organization">
    Go to [Settings](https://app.formabledocs.com/settings) and create an organization if you don't have one yet. You need an organization before you can generate keys or register webhooks.
  </Step>
  <Step title="Generate a key">
    In the **API keys** section, click **Generate key**, give it a name, and confirm.
  </Step>
  <Step title="Copy the key">
    Copy the full key immediately and store it as an environment secret. Formable shows it **once** — after you close the dialog, only a short prefix (for example `fmbl_a1b2c3d4...`) remains visible.
  </Step>
</Steps>

<Warning>
  Never expose API keys in browser or mobile code. Call the API from your backend, and only pass short-lived signing and redline URLs to the client.
</Warning>

You can rename or delete keys from the same Settings page. Deleting a key revokes it immediately. Each account can have up to 10 active keys.

## Use the key

Include the key in the `Authorization` header of every API request.

<CodeGroup>

```bash cURL
curl --request GET \
  --url https://api.formabledocs.com/api/v0/signature-requests \
  --header 'Authorization: Bearer fmbl_YOUR_API_KEY'
```

```javascript Node.js
const res = await fetch(
  "https://api.formabledocs.com/api/v0/signature-requests",
  {
    headers: {
      Authorization: `Bearer ${process.env.FORMABLE_API_KEY}`,
    },
  },
);
```

```python Python
import os
import requests

res = requests.get(
    "https://api.formabledocs.com/api/v0/signature-requests",
    headers={"Authorization": f"Bearer {os.environ['FORMABLE_API_KEY']}"},
)
```

</CodeGroup>

Keys start with `fmbl_`. Treat them like passwords: store them in a secrets manager, rotate by creating a new key and deleting the old one, and never commit them to source control.

## Base URL

```text
https://api.formabledocs.com/api
```

All endpoints are versioned under `/v0`, so a full URL looks like `https://api.formabledocs.com/api/v0/signature-requests`.

## Authorization errors

Requests fail with a `401` when the bearer token is missing, malformed, revoked, or expired:

```json
{
  "error": "Auth token not valid"
}
```

A `403` means the token is valid but your organization isn't authorized for the resource — for example, creating an edit URL for a template that belongs to another organization.
