The Formable API is organized around **templates** and **signature requests** for e-signing, with optional **redline request** endpoints for contract negotiation. It's a REST API with predictable, resource-oriented URLs, JSON request and response bodies, and standard HTTP status codes.

## Base URL

All endpoints are served under `/api` and versioned with `/v0`.

```text
https://api.formabledocs.com/api
```

## Authentication

Every endpoint except the health check requires a Bearer token in the `Authorization` header. Create an API key in [Settings](https://app.formabledocs.com/settings), then pass it on each request. See [Authentication](/authentication) for details.

```bash
Authorization: Bearer fmbl_YOUR_API_KEY
```

## Response format

Successful responses return JSON with a `200` status. Errors return a JSON object with a single `error` string:

```json
{
  "error": "templateId is required"
}
```

## Status codes

| Code | Meaning |
| --- | --- |
| `200` | Success. |
| `400` | Bad request — missing or invalid parameters. |
| `401` | Unauthorized — missing or invalid bearer token. |
| `403` | Forbidden — the token is valid but not authorized for this resource. |
| `404` | Not found — the referenced resource doesn't exist. |
| `409` | Conflict — the resource isn't in a valid state (for example, signing an already-completed document). |
| `413` | Payload too large — the uploaded file exceeds 40MB. |
| `500` | Internal server error. |
