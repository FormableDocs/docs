The Formable API lets you collect signatures inside your application without building the signing experience from scratch. Upload a document once as a **template**, send it as a **signature request**, and embed a short-lived signing URL in an iframe.

In simple terms, Formable API lets you setup a document to be signed and to get signed.

## What you can build

- **Embedded e-signature** — Send a document to a signer, drop the signing URL into an iframe, and retrieve the completed PDF when they're done.
- **Templates with prefilled fields** — Upload a document, place fields in the editor, then set values per signer at send time.

## How it works

<Steps>
  <Step title="Create a template">
    Upload a PDF or DOCX with [Create a template](/api-reference/endpoint/create-template). Formable returns a `templateId` you reuse for every send.
  </Step>
  <Step title="Create a signature request">
    Start a [signature request](/api-reference/endpoint/create-signature-request) from the template with a signer and sender.
  </Step>
  <Step title="Generate a signing URL">
    Request an embeddable [signing URL](/api-reference/endpoint/create-signing-url) and load it in an iframe in your product.
  </Step>
  <Step title="Retrieve the signed PDF">
    When signing finishes, fetch the [signed envelope](/api-reference/endpoint/get-signed-envelope), or listen for the [`document_signed`](/webhooks/events#document_signed) webhook.
  </Step>
</Steps>
